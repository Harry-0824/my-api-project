## 1. 專案簡介 (Project Overview)

本專案旨在為 **MG Motor** 打造一個具備國際化質感的品牌官方網站。透過 React 與 Node.js 的全端整合，提供車主與潛在客戶完整的品牌體驗、車業資訊、即時最新消息以及流暢的線上預約互動功能。

## 2. 技術架構 (Technical Architecture)

根據目前的技術演進與專案需求，架構規劃如下：

- **前端框架 (Frontend)**: 採用 **React** 作為核心，搭配 **React Router** 進行路由管理。
- **樣式處理 (Styling)**: 全面採用 **Styled-components** 進行樣式封裝，以提升架構統一性並優化效能。
- **後端核心 (Backend)**: 基於 **Node.js** 與 **Express (v5.2.1)** 建構，採用 RESTful API 架構。
- **資料庫 (Database)**: 使用 **PostgreSQL** (託管於 **Supabase**)，並透過 **Sequelize ORM** 進行進階的模型關聯管理。
- **安全認證 (Security)**: 整合 **JWT (JSON Web Token)** 身分驗證與 **bcryptjs** 密碼加密技術，確保會員資料安全。
- **基礎設施 (Infrastructure)**:
  - **Node.js 版本**: >= 18.0.0
  - **環境變數**: 使用 `dotenv` 管理敏感資訊 (DB 密碼、JWT Secret)。
  - **自動化指令**: 內建 `db:seed` 腳本，實現初始資料一鍵填入。
  - **部署平台**: Google Cloud Run (生產環境)，透過 Supabase Transaction Pooler 連線 (port 6543)。

## 3. 核心功能與資料模組 (Core Modules)

| 功能模組       | 核心 API 支援   | 主要組件與技術重點                              |
| :------------- | :-------------- | :---------------------------------------------- |
| **首頁展示**   | `/api/slides`   | Carousel (動態輪播)、HomeSlide 模型控制         |
| **車系中心**   | `/api/models`   | HS/ZS 四大車系展示、圖片 Gallery                |
| **規格詳情**   | `/api/trims`    | 包含詳細 Trim 內容、1.5T/2.0T 動力規格參數      |
| **新聞與活動** | `/api/articles` | 文章標題、預覽圖、分類與發布日期過濾            |
| **會員/驗證**  | `/api/auth`     | 註冊/登入 (bcrypt 加密)、JWT Profile 獲取       |
| **預約試乘**   | 整合中          | BookingForm 互動表單、整合會員 API 紀錄預約資訊 |

## 4. 專案結構圖 (Project Directory Structure)

```text
my-api-project/
├── config/             # 資料庫連線與車型靜態資料 (vehicleData.js)
├── controllers/        # 核心業務邏輯 (Article, Auth, Vehicle... 控制器)

├── scripts/            # setup-db.js (資料庫初始化), seedAllData.js (大量填入測試資料)
├── tests/              # 完整的 http 測試檔案與 Postman Collection
└── index.js            # 伺服器入口點與 CORS 設定
```

## 5. 開發規範與後續規劃 (Roadmap)

1.  **資料庫遷移完成**: 已從 Cloud SQL (MySQL) 遷移至 Supabase (PostgreSQL)，驅動套件由 `mysql2` 換為 `pg`，Sequelize dialect 更新為 `postgres`，並啟用 SSL 連線。
2.  **資料原子化**: 目前已完成 HS 與 ZS 的 `VehicleTrim` 資料結構（包含動力、安全、內裝等細節），支援高度客製化的頁面渲染。
3.  **安全性增強**: 已導入 Middleware 機制，保護 `/profile` 等機敏端點；JWT Secret 已更換為高熵隨機字串。
4.  **自動化測試**: 提供完整的 `.http` 測試腳本，確保 API 在不同環境（Local/Production）的一致性。
5.  **下一階段目標**:
    - 對接前端 Booking 系統實體資料寫入。
    - 完成各車系細部規格頁面 (Detailed Specs) 的前端對接。
